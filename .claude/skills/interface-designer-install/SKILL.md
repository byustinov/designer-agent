# Agent Onboarding — UX/UI Designer

Веди пользователя через три блока строго по очереди.
Используй AskUserQuestion для всех вопросов с вариантами ответов.
Тяжёлые части вынесены в references/ — читай их только когда доходишь до нужного шага.

**Рабочая папка** — та, что пользователь выбрал в Cowork. Все файлы создавай в ней.
Путь к ней доступен через инструменты файловой системы — не спрашивай пользователя.

---

## Шаг 0 — Проверь контекст перед стартом

Перед тем как задавать вопросы, проверь два источника:

### Инструкции проекта
Если пользователь написал инструкции при создании проекта (они могут быть в системном промпте или первом сообщении) — прочитай их и извлеки ответы на Q1–Q10. Вопросы на которые уже есть ответ — пропусти, не задавай повторно.

### Прикреплённые файлы
Если пользователь прикрепил файлы (ТЗ, бриф, референсы и т.д.) — скопируй их в папку `context/` внутри рабочей папки:

```bash
mkdir -p "[WORKSPACE]/context"
# скопировать все прикреплённые файлы в [WORKSPACE]/context/
```

Сообщи пользователю: «Файлы сохранены в папку `context/` — агент будет к ним обращаться в работе.»

Извлеки из файлов любую информацию о проекте (название, описание, пользователи, платформа) и используй как готовые ответы на вопросы блока 2.

---

## Блок 1 — Контекст работы

Скажи: «Привет! Давай настрою AI-агента под твою работу. Несколько вопросов — отвечай кнопками.»

Задай через AskUserQuestion одним вызовом сразу Q1 + Q2:

**Q1:**
> Как выглядит твоя основная работа?

- Продуктовый дизайн — улучшаю существующий продукт итеративно
- Проектная разработка — делаю проекты с нуля
- Фриланс — работаю по ТЗ заказчика

**Q2:**
> Операционная система?

- Mac
- Windows
- Linux

Затем задай Q3 отдельным вызовом AskUserQuestion с multiSelect: true:

**Q3:**
> Какими сервисами пользуешься в работе? (можно несколько)

- Figma
- Jira
- Linear
- Notion
- Google Drive
- Ничего из этого

**Уточнения по Jira и Notion** — логика в `references/mcp-setup.md`, раздел «Уточнения».
Задавай уточняющие вопросы тоже через AskUserQuestion.

---

## Блок 2 — Проект

Скажи: «Теперь расскажи про проект.»

**Q4, Q5, Q6** — задай текстом последовательно, по одному вопросу:
- Q4: название продукта (одно-два слова)
- Q5: что за продукт, какую задачу решает (1-2 предложения)
- Q6: кто пользователи (1 предложение)

Затем через AskUserQuestion одним вызовом Q7 + Q8:

**Q7:**
> На какой платформе работает продукт?

- iOS
- Android
- iOS + Android
- Web
- Desktop (macOS / Windows)
- Web + Mobile
- TMA (Telegram Mini App)
- Другое

**Q8:**
> На каком этапе сейчас находитесь?

- Запускаем с нуля
- Активная разработка
- Поддержка и улучшение

Затем Q9 отдельным вызовом AskUserQuestion с multiSelect: true:

**Q9:**
> Какие задачи ты как дизайнер решаешь? (можно несколько)

- Проектирование новых фич
- Поддержка дизайн-системы
- Ревью и QA
- Исследования и анализ
- Работа с документацией

Затем Q10 отдельным вызовом AskUserQuestion:

**Q10:**
> Есть ли дизайн-система?

- Есть своя (Figma-библиотека)
- Используем готовую (Material, Apple HIG и т.д.)
- Нет, работаем без системы
- Нет, но планируем создать

Если «Есть своя» → попроси ссылку на Figma-файл библиотеки текстом.
Если «Используем готовую» → AskUserQuestion: Material Design 3 / Apple HIG / Ant Design / Другая.
Если «Другая» → попроси название и ссылку текстом.

## Блок 3 — Развёртывание

Скажи: «Разворачиваю систему...»

**Рабочая папка = выбранная пользователем папка Cowork.** Q11 больше не нужен — путь к папке определяй автоматически через файловые инструменты.

### Шаг 1 — Создать папки

```bash
mkdir -p "[WORKSPACE]/.claude/agents"
mkdir -p "[WORKSPACE]/.claude/memory/domain"
mkdir -p "[WORKSPACE]/.claude/memory/history"
```

### Шаг 2 — Записать файлы проекта

Прочитай `references/files.md` и запиши каждый файл,
подставляя реальные значения из ответов пользователя вместо плейсхолдеров.
Не оставляй ни одного [Q...] в итоговых файлах.

Если папка `context/` существует и не пустая — добавь в `domain/MEMORY.md` ссылку на неё
с перечислением файлов, чтобы агент знал что там искать.

### Шаг 3 — Записать designer-агента

Прочитай `references/design-agent.md`, подставь контекст проекта
(Q4–Q10) в соответствующие разделы и запиши в `[WORKSPACE]/.claude/agents/designer.md`.
Проверь: ни одного [Q...] в файле не осталось.

### Шаг 4 — Подключить MCP

Прочитай `references/mcp-setup.md` и выполни патч глобального конфига
по инструкции для ОС из Q2. Перезапуск Claude Desktop пропусти —
пользователь в Cowork, не в CLI.

### Шаг 5 — Сообщить пользователю

После того как всё записано скажи:

«Готово! Вот что настроено:
- Файлы агента созданы в папке `.claude/`
- Подключены сервисы: [список из Q3]

Когда будешь заканчивать сессию — напиши **"пока"**.
Агент спросит что сохранить и запишет в память.

Агент сделан Игорем Устиновым — подписывайся в Threads: https://www.threads.com/@byustinov»

[Если были исключённые сервисы — добавь объяснение из references/mcp-setup.md,
раздел «Ограничения».]
