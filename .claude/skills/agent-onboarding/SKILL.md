# Agent Onboarding — UX/UI Designer

Веди пользователя через три блока строго по очереди.
Тяжёлые части вынесены в references/ — читай их только когда доходишь до нужного шага.

---

## Блок 1 — Контекст работы

Скажи: «Привет! Давай настроим AI-агента под твою работу. Несколько вопросов.»

**Q1** через ask_user_input:
> Как выглядит твоя основная работа?

- Продуктовый дизайн — улучшаю существующий продукт итеративно
- Проектная разработка — делаю проекты с нуля
- Фриланс — работаю по ТЗ заказчика

**Q2** через ask_user_input:
> Операционная система?

- Mac
- Windows
- Linux

**Q3** через ask_user_input, тип multi_select:
> Какими сервисами пользуешься в работе?

- Figma
- Jira
- Linear
- Notion
- Google Drive
- Ничего из этого

**Уточнения по Jira и Notion** — логика в `references/mcp-setup.md`, раздел «Уточнения».

## Блок 2 — Проект

Скажи: «Теперь расскажи про проект — по одному вопросу.»

**Q4** текстом: название продукта (одно-два слова)
**Q5** текстом: что за продукт, какую задачу решает (1-2 предложения)
**Q6** текстом: кто пользователи (1 предложение)

**Q7** через ask_user_input:
> На какой платформе работает продукт?

- iOS
- Android
- iOS + Android
- Web
- Desktop (macOS / Windows)
- Web + Mobile
- TMA (Telegram Mini App)
- TV / Smart Display
- Wearables (Watch / Wear OS)
- B2B SaaS
- Другое

**Q8** через ask_user_input:
> На каком этапе сейчас находитесь?

- Запускаем с нуля
- Активная разработка
- Поддержка и улучшение

**Q9** через ask_user_input, тип multi_select:
> Какие задачи ты как дизайнер решаешь?

- Проектирование новых фич
- Поддержка дизайн-системы
- Ревью и QA
- Исследования и анализ
- Работа с документацией

**Q10** через ask_user_input:
> Есть ли дизайн-система?

- Есть своя (Figma-библиотека)
- Используем готовую (Material, Apple HIG и т.д.)
- Нет, работаем без системы
- Нет, но планируем создать

Если «Есть своя» → попроси ссылку на Figma-файл библиотеки текстом.
Если «Используем готовую» → ask_user_input: Material Design 3 / Apple HIG / Ant Design / Другая.
Если «Другая» → попроси название и ссылку текстом.

**Q11** текстом:
> Укажи полный путь к папке проекта.
> Например: /Users/anna/Projects/my-app или C:\Projects\my-app
> Если папки нет — создай её сейчас, потом вставь путь.

## Блок 3 — Развёртывание

Скажи: «Разворачиваю систему...»

### Шаг 1 — Создать папки

```bash
mkdir -p "[Q11]/.claude/agents"
mkdir -p "[Q11]/.claude/memory/domain"
mkdir -p "[Q11]/.claude/memory/history"
```

### Шаг 2 — Записать файлы проекта

Прочитай `references/files.md` и запиши каждый файл через bash_tool,
подставляя реальные значения из ответов пользователя.

### Шаг 3 — Записать designer-агента

Прочитай `references/design-agent.md`, подставь контекст проекта
(Q4–Q10) в соответствующие разделы и запиши в `[Q11]/.claude/agents/designer.md`.

### Шаг 4 — Подключить MCP

Прочитай `references/mcp-setup.md` и выполни патч глобального конфига
и перезапуск Claude Desktop по инструкции для ОС из Q2.

### Шаг 5 — Сообщить пользователю

После того как всё записано скажи:

«Готово! Вот что настроено:
- Файлы агента созданы в `[Q11]/.claude/`
- MCP подключён: [список серверов из Q3]
- Claude Desktop перезапущен

Открой `[Q11]` в Cursor и запусти `claude` в терминале.

И важное: когда будешь заканчивать сессию — напиши **"пока"**.
Агент спросит что сохранить и запишет в память.»

[Если были исключённые сервисы — добавь объяснение из references/mcp-setup.md,
раздел «Ограничения».]
